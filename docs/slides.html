OpenTama

出社促進 × 知見伝播キット

Tamagotchi x Office WiFi x Knowledge Transfer

いま OpenTama ができること、ひとことで言うと

WiFi に繋ぐだけで、
KT (Knowledge Transfer) が
勝手に走る。

出社する → ペットが育つ → すれ違った人と挨拶する → その日の出会いが Teams に流れる。
社員は 何もボタンを押さなくていい。

全体像

5 つの層がつながって、KT のパイプラインを作っている。

① 社内 WiFi 検知
→
② Takoron 成長
→
③ 赤外線すれ違い
→
④ Teams ダイジェスト投稿
飼い主は ⓪ Claude Code Skill として呼び出すだけ (python -m opentama status 相当)

① と ② は時間で勝手に進む。③ は近接時に勝手に走る。
④ は決まった時刻に勝手に投稿される。
人間の介入ポイントは「朝、PC を開く」だけ。

4 つの端点 × 送受信

同じ OpenTama プロトコル ("OT" | ver | type | len | payload | crc16) を 4 つの端末が共有。

端点 送信 (TX) 受信 (RX) 備考
PC (Python skill) ✓ ✓ USB-IR / Loopback / IrDA
M5StickC Plus2 ✓ ✓ 内蔵 IR LED + Grove IR Unit
実物のガラケー ✓ ✓ IrDA vCard + OBEX PUT
Microsoft Teams ✓ — Power Automate Webhook 出力のみ

Teams 入力（Bot Framework）は Roadmap。残りは既に動く。

① 社内 WiFi で成長する

仕組み

  • OS の SSID を netsh / iw / airport で取得
  • 設定済み company_ssid と一致するときだけ成長点 (gp) が貯まる
  • 離席するとペットの機嫌が 4 倍速で下がる
  • 状態は ~/.opentama/state.json に保存

進化ステージ

  • egg → 0 gp
  • baby → 10 gp
  • child → 50 gp
  • teen → 200 gp
  • adult → 500 gp
  • elder (王さま) → 1500 gp 👑

⓪ Claude Code Skill として動く

OpenTama は SKILL.md 付きの Claude Code skill。Claude が自然言語で呼び出す。

トリガー語句の例

  • 「出社した」「office に着いた」 → status 自動実行 → 成長 tick
  • 「たまごっち の様子は?」 → status --display iro
  • 「Teams で報告して」 → teams notify
  • 「今日のすれ違い」 → proximity digest --notify-teams
  • 「IR で待ち受け」 → proximity scan --port serial://...
  • 「vCard を broadcast」 → proximity broadcast

朝、Claude に「出社した」と言うだけで、status 更新 + proximity scan 開始 + Teams 報告まで一気通貫。

③ 赤外線で すれ違い 検知

独自の小さなフレームプロトコル。PC / M5StickC が同じバイト列を読み書きできる。

+---------+---------+--------+------------------+---------+--------+
| "OT"    | version | type   | payload length   | payload | crc16  |
| 2 bytes | 1 byte  | 1 byte | 2 bytes (uint16) | N bytes | 2 bytes|
+---------+---------+--------+------------------+---------+--------+

   type:  HELLO / STATE / GIFT / VISIT / ACK
   CRC:   CCITT-FALSE (poly=0x1021, init=0xFFFF)
   payload: JSON   {"name": "...", "stage": "...", "gp": ...}

実物のガラケー (IrDA) は別レイヤーで vCard / vNote として受信。 M5StickC ⇄ PC ⇄ ガラケー が同じ proximity log に集約される。

3 つの「ガラケー」が同じ世界で会話する

A. PC + USB-IR adapter

標準。serial:///dev/ttyUSB0 経由でフレーム送受信。 proximity scan / broadcast / IR session が全部ここに乗る。

B. M5StickC Plus2 (TX + RX)

ESP32 firmware (firmware/m5stickc/) が IR LED で HELLO を吐き、Grove の IR Unit で他の peer を受信。 LCD にガラケー風の最新ピア名を表示。

C. 実物のガラケー (IrDA)

受信: 携帯から「名刺送信」→ PC の proximity scan --garake が vCard を取り込む。
送信: PC の proximity broadcast が OBEX PUT を投げ続け、携帯は受信スタンバイのまま自動受信。

④ Teams に出会いが流れる

仕組み

  • Power Automate の "When a Teams webhook request is received" ワークフロー
  • Adaptive Card 1.4 を JSON で POST するだけ
  • Microsoft Graph も OAuth も不要
  • Webhook URL は環境変数 OPENTAMA_TEAMS_WEBHOOK

投げられるカード

  • status card — ペットの状態スナップショット (name, stage, stats, achievements)
  • digest card — 「今日 N 人とすれ違いました」と peer 一覧

proximity digest --notify-teams がパイプラインの仕上げ。

「KT が勝手に走る」一日

飼い主の操作は 朝に PC を開いた その一回だけ。

09:00
たけしさん出社。Claude が「出社しました」を検知 → status → 社内 WiFi 確認 OK → ペット たけ太郎 成長開始。
09:15
給湯室で りえこさんとすれ違い。
M5StickC 同士が IR で HELLO 交換 → 両方の proximity log に peer=りえこ を追記。
10:30
会議室前で ともみさんのガラケー (赤外線受信スタンバイ) とすれ違い。
PC の proximity broadcast が vCard 「たけ太郎」を OBEX PUT → 自動で携帯に届く。
12:00
昼休み。M5StickC の LCD に RX: りえこ (#1) が浮かぶ。 「あ、午前中 りえこさん いたんだ」と気づく。
17:00
Claude が proximity digest --notify-teams を実行 → Teams チャンネルに Adaptive Card が自動投稿:
「📡 たけしさんは 今日 2 人とすれ違いました — りえこ / ともみ」
→ たけしさん「りえこさん kubernetes 詳しいって言ってたな、明日聞こう」
これが KT。

python -m opentama status --display iro

$env:OPENTAMA_TEAMS_WEBHOOK = "https://prod-XX..."
python -m opentama teams notify

python -m opentama proximity scan --port serial:///dev/ttyUSB0 --duration 60
python -m opentama proximity digest --notify-teams
